# ResNet: Deep Residual Learning for Image Recognition

> **핵심 요약:** "층을 더 쌓으면 성능이 좋아질 것"이라는 상식이 현실에서
> 무너지는 문제(Degradation)를 발견하고, **잔차 연결(Skip Connection)**이라는
> 단순한 아이디어 하나로 152층짜리 초거대 신경망을 성공적으로 학습시킨
> 기념비적인 논문입니다. ILSVRC 2015 대회에서 5개 부문 전체 1위를 달성했습니다.

------------------------------------------------------------------------

## 1. 개요 및 배경 (Background)

### 왜 이 논문이 나왔는가?

AlexNet(2012) 이후 딥러닝 커뮤니티는 자연스럽게 이런 결론을 내렸습니다.
**"층이 깊을수록 성능이 좋아지니까, 더 깊이 쌓으면 되겠다."**

그런데 실제로 실험해보니 이상한 일이 벌어졌습니다. 56층짜리 네트워크가
20층짜리보다 훈련 오류(Training Error)가 **더 높게** 나온 겁니다.
이건 과적합(Overfitting) 때문이 아니었습니다. 오히려 학습 자체가 제대로
안 되고 있다는 신호였습니다.

논문 저자들(Kaiming He 외 3명, Microsoft Research)은 이 현상에
**"Degradation Problem(성능 저하 문제)"** 이라는 이름을 붙이고,
그 원인과 해결책을 파고들었습니다.

### 기존 방식의 문제점

1.  **기울기 소실(Vanishing Gradient):** 층이 깊어질수록, 학습 신호가
    앞쪽 층까지 전달되는 과정에서 점점 작아지다가 사라집니다. 제일 앞쪽
    층들은 사실상 아무것도 배우지 못하는 상태가 됩니다.
2.  **Degradation Problem(성능 저하 문제):** 기울기 소실과는 별개로,
    층을 그냥 쌓기만 해도 깊어질수록 오히려 정확도가 떨어지는 현상이
    발생합니다. 이상적으로는 "아무것도 안 하는 층(항등 함수, Identity
    Mapping)"이라도 추가하면 최소한 성능이 유지돼야 하는데, 실제로는
    신경망이 그 단순한 항등 함수조차 제대로 학습하지 못했습니다.
3.  **최적화의 어려움:** 층이 많아질수록 손실 함수의 형태가 복잡해져서
    어디로 가야 할지(최적값) 찾기가 점점 어려워집니다.

### ResNet의 핵심 해결책 (Solution)

-   **잔차 학습(Residual Learning):** 층이 "정답"을 통째로 배우는 대신,
    "입력에서 얼마나 바꿔야 하는가(잔차)"만 배우도록 구조를 바꿨습니다.
-   **Skip Connection(지름길 연결):** 입력값을 몇 개의 층을 건너뛰어
    출력에 직접 더해주는 구조를 만들었습니다. 이 경로 덕분에 기울기가
    소실 없이 앞쪽까지 곧장 전달됩니다.
-   **초깊은 네트워크 실현:** 위 두 가지 덕분에 152층짜리 네트워크를
    안정적으로 학습시키는 데 성공했습니다.
-   **Batch Normalization 적극 활용:** 각 층의 입출력을 정규화해서
    학습을 안정시켰습니다.

## 2. 핵심 구조 및 기술 상세

### ① Residual Learning (잔차 학습) — 이 논문의 핵심 아이디어

-   **비유:** 시험 공부를 할 때, 처음부터 전부 다 외우려 하지 말고
    "내가 틀린 부분만 집중적으로 고치자"는 전략입니다.
-   **기존 방식:** 신경망이 목표 출력 H(x)를 직접 통째로 학습합니다.
-   **ResNet 방식:** 신경망이 H(x)를 바로 배우는 대신, **F(x) = H(x) - x**,
    즉 "입력에서 얼마나 달라져야 하는지(잔차)"만 학습합니다. 최종 출력은
    **F(x) + x**가 됩니다.
-   **왜 이게 유리한가?** 만약 어떤 층이 아무것도 안 해도 되는 상황이라면,
    F(x)를 0으로 만들기만 하면 됩니다. 이게 H(x) = x(항등 함수)를
    통째로 학습하는 것보다 훨씬 쉽습니다.

```
수식: Output = F(x) + x
      (F(x): 층이 학습한 잔차 / x: Skip Connection으로 넘어온 원본 입력)
```

### ② Skip Connection / Shortcut Connection (지름길 연결)

-   **비유:** 고속도로에서 출구를 거치지 않고 목적지까지 바로 가는
    지름길 도로입니다.
-   **구조:** 입력 x가 Residual Block 안쪽 층들을 거치는 경로와,
    층들을 그냥 건너뛰어 출력에 직접 더해지는 경로, 이 두 경로가
    병렬로 존재합니다.
-   **두 가지 방식:**
    -   **Identity Shortcut:** 입력과 출력의 크기가 같을 때, x를 그대로
        더합니다. 추가 파라미터가 전혀 없습니다.
    -   **Projection Shortcut:** 입력과 출력의 채널 수나 크기가 다를 때,
        1×1 합성곱으로 크기를 맞춰준 뒤 더합니다.
-   **핵심 효과:** 기울기가 역방향으로 전파될 때 이 지름길을 통해 소실
    없이 앞쪽 층까지 직접 전달됩니다.

### ③ Residual Block (잔차 블록) — 기본 구조 단위

-   **비유:** 레고 블록 하나. 이 블록을 여러 개 쌓아서 전체 네트워크를
    만듭니다.
-   **Basic Block (ResNet-18, 34에서 사용):**
    -   [Conv 3×3 → Batch Norm → ReLU → Conv 3×3 → Batch Norm] + Skip 연결 후 → ReLU
    -   2개의 합성곱 층으로 구성된 기본형입니다.
-   **Bottleneck Block (ResNet-50, 101, 152에서 사용):**
    -   [Conv 1×1(채널 줄이기) → Conv 3×3(특징 추출) → Conv 1×1(채널 복원)] + Skip 연결
    -   층이 더 깊어질수록 계산량이 폭발하는 걸 막기 위해, 1×1 합성곱으로
        채널 수를 먼저 줄인 뒤 3×3 합성곱을 적용하고 다시 복원합니다.
        병목(Bottleneck)처럼 중간이 좁아지는 구조라서 붙은 이름입니다.

### ④ Batch Normalization (배치 정규화)

-   **비유:** 팀 프로젝트에서 각 팀원이 제출한 자료의 형식이 제각각이면
    취합이 힘듭니다. 제출 전에 모두 통일된 양식으로 맞추는 작업입니다.
-   **내용:** 각 층의 입출력 값 분포를 평균 0, 분산 1에 가깝게 정규화합니다.
    이렇게 하면 학습이 안정되고, 더 높은 학습률(Learning Rate)을 사용할
    수 있어 학습이 빨라집니다.
-   **위치:** ResNet에서는 매 합성곱 층 직후에 적용됩니다.
-   **추가 효과:** Dropout 없이도 과적합을 어느 정도 억제해주는 효과가 있습니다.

### ⑤ Global Average Pooling (전역 평균 풀링)

-   **비유:** 사진 전체를 하나의 숫자로 압축하는 것처럼, 특징 지도(Feature Map)
    전체를 채널별로 평균 내어 하나의 값으로 줄입니다.
-   **내용:** AlexNet에서 사용하던 Fully Connected Layer 여러 개 대신,
    ResNet은 마지막에 Global Average Pooling 하나만 씁니다.
-   **효과:** 파라미터 수를 대폭 줄여 과적합을 억제하고, 입력 이미지
    크기가 달라져도 유연하게 대응할 수 있습니다.

### ⑥ He Initialization (He 초기화)

-   **비유:** 마라톤 출발선에서 선수들의 시작 위치를 잘 배분하는 것처럼,
    신경망 가중치의 초깃값을 잘 설정하는 방법입니다.
-   **내용:** ReLU 함수를 사용하는 깊은 네트워크에 최적화된 가중치 초기화
    방식입니다. 이 논문의 1저자 Kaiming He가 제안한 방법으로, 층이 매우
    깊어도 처음부터 학습이 안정적으로 시작될 수 있게 해줍니다.

## 3. 전체 아키텍처 변형 비교

  모델         층 수   블록 종류         파라미터 수    특징
  ------------ ------- ----------------- -------------- -------------------------
  ResNet-18    18층    Basic Block       1,100만 개     가장 가볍고 빠름
  ResNet-34    34층    Basic Block       2,100만 개     균형 잡힌 입문 모델
  ResNet-50    50층    Bottleneck Block  2,500만 개     실무 기본값, 가장 많이 씀
  ResNet-101   101층   Bottleneck Block  4,400만 개     높은 정확도
  ResNet-152   152층   Bottleneck Block  6,000만 개     논문 최고 성능, VGG의 8배 깊이

> **참고:** ResNet-50부터는 계산량 절감을 위해 Bottleneck 블록으로 전환됩니다.

### 층 깊이별 성능 비교 (Plain Net vs ResNet)

  모델 종류         층 수   특이점
  ----------------- ------- -----------------------------------------------
  Plain Network     20층    정상적으로 학습됨
  Plain Network     56층    20층보다 오히려 성능 나쁨 (Degradation 확인)
  ResNet            20층    Plain 20층과 유사
  ResNet            56층    Plain 56층보다 훨씬 좋음 (문제 해결 확인)
  ResNet            152층   성능이 계속 향상됨

## 5. 이후에 미친 영향

ResNet이 발표된 후, Skip Connection 개념은 딥러닝 전반으로 퍼져나갔습니다.

-   **DenseNet(2016):** 모든 층을 이후 모든 층에 연결하는 더 극단적인 형태로 발전
-   **Transformer(2017):** BERT, GPT 같은 언어 모델 구조에도 Residual Connection이
    필수 요소로 채택됨
-   **EfficientNet, ResNeXt 등:** ResNet 기반의 다양한 개선 모델 등장
-   **AlphaFold, AlphaGo Zero:** 과학·게임 AI 분야에도 핵심 구조로 채택

------------------------------------------------------------------------

## 핵심 정리

ResNet은 "층을 무한정 쌓으면 성능이 좋아진다"는 믿음이 현실에서 무너지는
문제를 직시하고, Skip Connection이라는 우아하고 단순한 아이디어로 해결한
모델이다. 이 아이디어는 이후 CNN을 넘어 AI 전반의 기본 설계 원칙이 되었다.
